# 《幻界迴響：霓虹傳說》- 遊戲系統企劃與原型展示

這是一個結合個人對戰術 RPG 體系的理解，嘗試進行系統化規劃的練習專案。我將心中對於二次元遊戲的熱忱與想法，透過互動原型的實作，試著梳理成可供開發參考的藍圖。

## 遊戲企劃核心：世界觀與定位

### 世界觀：異界疊影

本專案為二次元 RPG 新作《幻界迴響：霓虹傳說》的沉浸式宣傳站台。透過雙維度（現實 vs 異界）的視覺對比，將玩家代入「導標者」的敘事宿命。

## 核心賣點 (USPs)

### 1. 戰術共鬥策略
戰鬥採用 4+1 陣容（4 名角色+1 名援護角）。透過「連鎖反應系統」，玩家可透過穿梭現實與異界採取不同策略，在保留回合制深度的同時，導入 MMO 式的共鬥機制。

### 2. 都市社交場域
不僅是虛擬空間，玩家可在都市中經營店鋪、舉辦虛擬偶像演唱會，將社交與課金點（外觀、家具）深度連結，凝聚長線社群力。

### 3. 敘事級長線營運
採用章節式更新，每一週期的重大版本均會帶來全新的都市區域與異世界地圖，保持玩家的高黏著度與探索渴望。

## 進階系統設計 (Extended Design)

* **永續平衡數值體系**：拒絕數值膨脹，透過精確的底層公式設計，確保初期角色在後續版本中仍具備功能性與搭配價值。

* **羈絆迴響敘事**：透過好感度與專屬劇情，揭示「維度坍塌」的隱藏真相，強化玩家的敘事代入感。

* **精密武裝系統**：徹底捨棄隨機刷詞條，裝備屬性固定，將核心重心回歸戰術配置與角色技能組合。

* **維度共鳴彩蛋**：特定角色組合將觸發隱藏 Bonus 與特殊互動彩蛋，增加社群間的討論熱度。

* **智略討伐 (Multi-Solution Raids)**：Boss 戰設計至少三種破敵方案（連鎖控制、極速破防、持續反擊），鼓勵玩家深度融合策略思考。

## 開發技術棧與工作流 (Development Workflow)

為了能將企劃邏輯具體化，我運用了現代化工具鏈進行高效開發：

* **前端架構**：Vue3 / Vuetify

* **AI 輔助開發**：運用 Gemini (Canvas)、Generative AI 與 AI Agent 進行規格優化與原型邏輯檢查。

* **美術設計流程**：採用精準的 Prompt Engineering，依序產出：線稿 → 上色 → 光影 → 構圖 → 角色特徵 → 五官定義，確保視覺高度統一。

* **審核紀錄**：開發過程包含 `Agent.md` 審核企劃與 `OPTIMIZATION_LOG.md` 日誌追蹤。

## 專業學習與產出

* **企劃基礎**：持續練習 PRD 撰寫、戰鬥機制平衡、流程圖 (Flowchart) 繪製與數值模型規劃。

* **技術視角**：具備開發背景，因此能更謹慎地思考企劃需求與實作間的對接，力求產出的規格具備開發可行性。

* **熱忱**：長期關注傳奇網路的開發動態，對於遊戲 UI 互動與社群經營模式抱持高度學習熱忱，希望能有機會向產業前輩請益。
